# PJH-RPG

Realm of the Mad God 스타일의 웹 브라우저 2D 도트 MMORPG.
탑다운 뷰, 실시간 멀티플레이, 무기별 공격(근접 스윙/원거리 화살), 몬스터, 레벨/인벤토리/아이템/상점 시스템.

**이 프로젝트는 픽셀아트 기반 게임입니다.** 지금은 실제 도트 스프라이트 대신 Phaser Graphics로
찍은 색상 도형 placeholder를 쓰고 있지만(추후 실제 픽셀아트 에셋으로 교체 예정), 렌더링은 이미
`pixelArt: true`(nearest-neighbor, 블러 없는 각진 확대)로 설정해뒀습니다.

## 기술 스택

- **클라이언트**: [Phaser 3](https://phaser.io/) (CDN 로드, 번들러 없음, 순수 ES Module)
- **서버**: Node.js + Express + [Socket.io](https://socket.io/)
- **저장**: 브라우저 `localStorage` (계정/로그인 없음, 기기별로 캐릭터 저장)

## 실행 방법

```
npm install
npm run dev
```

`http://localhost:3000` 을 여러 브라우저 창(또는 탭)으로 열면 서로 다른 플레이어로 접속되어
실시간으로 위치가 동기화됩니다.

## 조작

- 이동: WASD / 방향키
- 조준/공격: 마우스 커서 방향, 클릭으로 공격 (장착 무기가 검이면 근접 스윙, 활이면 화살 발사)
- 화면 확대/축소: 마우스 휠 (1배~3배, 기본 2배로 줌인된 상태로 시작)
- 인벤토리 열기/닫기: **E** (또는 Esc로 닫기), 마인크래프트처럼 모달로 열림
- 상점 열기: 좌상단 HUD의 "상점 열기" 버튼 (**상점 텐트 근처에 있을 때만 활성화**)
- 관리자(베타 디버그) 패널: 우하단 "⚙ 관리자 (베타)" 버튼

## 유저 데이터 구조

캐릭터는 계정 시스템 없이 브라우저 `localStorage`(키: `pjh-rpg-character`)에 저장되고,
접속 시 서버로 전송되어 서버 쪽 `players[socket.id]`에 병합됩니다.

```js
{
  playerId: "uuid",          // 클라이언트가 최초 생성, 기기 내에서 캐릭터 식별용
  level: 1,                  // 1~10 (시작의 섬 레벨 캡)
  xp: 0,                     // 다음 레벨까지 누적 경험치 (레벨업 시 초과분만 이월)
  hp: 20,
  maxHp: 20,                 // 레벨에 따라 자동 계산 (shared/leveling.js maxHpForLevel)
  gold: 0,
  inventory: [               // 고정 12슬롯, 빈 슬롯은 null (앞 5칸은 하단 핫바에도 항상 표시)
    { itemId: "health_potion", qty: 3 },
    null, null, /* ... */
  ],
  equipped: { weapon: "wooden_sword", armor: null },
}
```

서버 쪽 플레이어 상태는 여기에 `x, y, rotation, color`(위치/외형, 저장 대상 아님)가 추가된 형태입니다.

## 맵 설계 — 시작의 섬 (레벨 1~10)

- `shared/islandMap.js`가 160×160 타일(5120×5120px) 섬을 절차적으로 생성합니다. 섬 중심에서의
  거리로 지형을 결정: 내륙은 잔디/흙(의사난수로 흙 patch 흩뿌림), 해안 5타일은 모래, 섬 반지름
  밖은 전부 바다. (이전엔 50×50이라 카메라가 거의 안 움직였는데, 면적 기준 약 10배로 키움)
- 서버가 한 번만 생성해서 클라이언트에 그대로 전송 — 생성 로직은 서버에만 있고, 클라이언트는 받은
  배열을 렌더링/충돌/미니맵에만 사용합니다.
- **바다 진입 제한**: `LEVEL_REQUIRED_FOR_SEA = 10` 기준으로
  - 레벨 10 미만: 바다 타일에 Arcade Physics 충돌이 걸려 물리적으로 진입 불가
  - 레벨 10 이상: 진입은 가능하지만 이동 속도가 `WATER_SPEED_MULTIPLIER(0.5)`로 감소 (배 없이 헤엄치는 느낌)
  - 이 판정은 레벨이 바뀔 때마다(`applyWaterCollision`) 다시 계산됩니다.
- 타일 그래픽은 아직 실제 도트 스프라이트가 아니라 색상 사각형 placeholder입니다
  (잔디=초록, 흙=갈색, 모래=베이지, 바다=파랑) — 도트 스프라이트 전환은 이후 과제로 미룸.
- **풀-흙 경계 블렌딩**: 흙 patch는 4×4 타일 블록 단위로 뭉쳐서 생성되고(`shared/islandMap.js`
  `applyGrassDirtBlend`), 서로 다른 타입과 맞닿은 타일은 `GRASS_EDGE`/`DIRT_EDGE`라는 시각 전용
  타일로 바뀌어 클라이언트에서 반점(디더) 무늬로 렌더링됩니다(`GameScene.drawDitherTile`) — 각진
  사각형 경계 대신 부드럽게 이어지는 것처럼 보이게 하는 용도로, 충돌/이동 판정에는 영향 없음.

## 자연물 오브젝트 (엄폐/무빙)

- 나무·바위·수풀 160개가 섬 전체(상점 반경 4타일 안쪽은 제외)에 흩뿌려져 있고, 정적
  Arcade Physics 바디로 플레이어 이동을 막습니다 — RotMG 특유의 "장애물 사이를 누비는 무빙"을
  위한 엄폐물입니다.
- 화살도 오브젝트에 막힙니다(서버 tick에서 화살 위치가 `OBSTACLE_RADIUS(14px)` 안에 들어오면
  소멸). 근접 공격은 오브젝트를 고려하지 않음(장애물 뒤 몬스터도 사거리 안이면 맞음 — 시야선
  차단은 미구현).
- 한 번 생성되면 사라지지 않는 정적 데이터라 실시간 이벤트 없이 `init` 페이로드에만 담아 보냅니다.

## 캐릭터 / 몬스터 / 상점 외형

- 원 하나짜리 placeholder 대신, 몸통(타원) + 머리(원) + 눈으로 구성된 탑다운 사람 실루엣
  (`GameScene.makeCharacterTexture`). 회전하면 머리가 바라보는 방향이 그대로 보여서 조준 방향을
  파악할 수 있음(원일 때는 회전해도 겉보기 변화가 없었음).
- 몬스터는 종류별로 색만 다른 뿔 달린 실루엣(`makeMonsterTexture`)이라 플레이어와 한눈에 구별됨.
- 플레이어/몬스터 발밑에 옅은 반투명 그림자 타원을 깔아서 공중에 떠 있는 느낌을 없앴습니다.
- 캐릭터·몬스터·나무/바위/수풀은 모두 자신의 y좌표를 depth로 써서(Y-sorting) 화면 아래쪽에
  있을수록 위에 그려집니다 — 나무 뒤로 걸어 들어가면 가려지는 식.
- 상점은 노란 사각형 대신 줄무늬 시장 텐트 모양 스프라이트(`makeShopTexture`)로 바뀌었습니다.
- 아직 색상 도형 placeholder이고, `pixelArt: true`라 확대돼도 흐려지지 않고 각지게 보임.

## 시각 효과(VFX)

- **근접 스윙**: 이전보다 훨씬 큰 칼자국(길이 46px, 사거리와 비슷하게 맞춤)이 조준 방향 기준
  좌우로 부채꼴을 그리며 사라짐.
- **화살 잔상**: 화살이 날아가는 동안 30ms마다 옅고 투명한 잔상을 남겨 궤적이 잘 보이게 함.
- **몬스터 피격 플래시**: 데미지를 받으면 몬스터 스프라이트가 80ms간 흰색으로 번쩍임
  (`setTintFill` → `clearTint`).

## 전투 — 무기별 공격 모션

- 무기 카탈로그의 `attackType`이 공격 방식을 결정합니다: `melee`(검류)는 근접 스윙,
  `ranged`(활)는 화살 발사. 아무 무기도 없으면 맨손 근접 공격(최소 데미지)으로 취급.
- **근접**: 클릭 시 서버가 즉시 판정 — 공격자 위치에서 반경 `MELEE_RANGE(46px)`, 조준 방향 기준
  좌우 45도(`MELEE_ARC`) 안에 있는 몬스터에게 무기 데미지를 입힘. 모든 클라이언트가 칼자국
  스윙 애니메이션(`playMeleeSwing`)을 함께 봄(공격자 본인 화면에서만 보이는 게 아님).
- **원거리**: 클릭 시 서버가 화살을 생성하고 발사 이벤트만 브로드캐스트 — 각 클라이언트는 같은
  속도(`ARROW_SPEED`)로 화살을 로컬 시뮬레이션해서 그리고, 서버는 내부적으로(화면에 안 보이는)
  같은 계산을 100ms tick마다 돌려 몬스터 명중 여부를 판정합니다. 맞으면 `arrowRemoved`를 쏴서
  모든 클라이언트에서 화살을 즉시 지우고 데미지를 적용합니다.
- 몬스터가 죽으면 기존 월드 아이템 픽업 시스템을 재사용해 그 자리에 XP 조각(항상)과 골드 더미
  (50% 확률)를 드랍하고, 10초 후 섬 어딘가에 같은 종류로 리스폰됩니다.
- 몬스터는 스폰 지점 주변을 배회만 할 뿐 플레이어를 공격하지 않습니다 — 플레이어 피격/사망/리스폰은
  아직 없음(알려진 한계 참고).

## 몬스터 (지역별 스폰)

- `MONSTER_TYPES`에 3종이 섬 중심에서의 거리(타일 반경)에 따라 나뉘어 스폰됩니다 — 마을에서
  멀어질수록 강해지는 구조:
  - 슬라임 (반경 0~25, HP 15, 12마리) — 마을 주변 초보 구간
  - 늑대 (반경 25~45, HP 30, 10마리) — 중간 지역
  - 게 (반경 45~섬 끝, HP 45, 8마리) — 해안가 외곽
- 서버가 100ms마다 도는 tick(`tickMonsters`)에서 각 몬스터를 스폰 지점 반경 150px 안에서
  2~5초 간격으로 새 목표 지점을 골라 배회시키고, 그 위치를 모든 클라이언트에 주기적으로
  브로드캐스트(`monstersUpdated`)합니다. 클라이언트는 받은 목표 위치로 부드럽게 보간합니다.

## 레벨 / 아이템 / 인벤토리 / 상점 시스템

- XP 곡선과 레벨업 처리(`applyXp`)는 서버(`shared/leveling.js`)가 authoritative하게 수행합니다.
  레벨업 시 최대 HP가 늘고 풀피로 회복됩니다.
- 아이템 카탈로그(`shared/items.js`): 무기(나무 검/철검/짧은 활), 방어구(가죽 갑옷), 소비 아이템
  (체력 물약, 스택 가능), 각 아이템에 상점 가격(`price`)과 무기는 `attackType`(melee/ranged)이 있음.
- 월드에는 세 종류의 픽업이 흩뿌려져 있고 걸어서 밟으면 자동 습득됩니다(다른 접속자 화면에서도
  즉시 사라짐 — 서버가 단일 진실 소스): 장비/물약(핑크), XP 조각(하늘색), 골드 더미(금색).
  습득한 자리는 8초 후 같은 종류로 리스폰됩니다. 몬스터를 처치해도 같은 방식으로 드랍됩니다.
- **인벤토리(마인크래프트 스타일)**: 하단에 항상 보이는 5슬롯 핫바 + **E키로 여닫는 전체 12슬롯
  모달**(장착 무기/방어구 표시 포함). 슬롯 클릭 시 소비 아이템은 사용(회복), 무기/방어구는 장착.
  모달이 열려 있는 동안은 이동/공격 입력이 멈춥니다.
- **상점(근접 제한)**: 섬 중앙 근처 상점 텐트에서 `SHOP_INTERACT_RADIUS(96px,
  약 3타일)` 안에 있어야만 HUD의 "상점 열기" 버튼이 활성화되고 구매/판매가 가능합니다. 서버도
  `buyItem`/`sellItem` 요청마다 플레이어의 실제 좌표로 거리 재검증을 하므로, 클라이언트를 조작해도
  멀리서는 살 수 없습니다. 상점 범위를 벗어나면 열려 있던 모달도 자동으로 닫힙니다.
- **미니맵**: 우상단에 섬 전체 지형을 축소해 표시하고, 내 위치(흰 점) / 다른 플레이어(회색 점) /
  상점(금색 점)을 실시간으로 보여줍니다.

## 관리자(베타 디버그) 패널

- 화면 우하단 "⚙ 관리자 (베타)" 버튼으로 골드/레벨을 자유롭게 입력해 즉시 적용할 수 있습니다
  (`adminSetGold`, `adminSetLevel` 이벤트). 레벨을 바꾸면 최대 HP도 재계산되고 풀피로 회복됩니다.
- **인증이 전혀 없습니다.** 지금 단계(베타 테스트)에서 빠르게 확인해보기 위한 용도이며, 정식
  서비스 전에는 반드시 제거하거나 서버 쪽 인증/권한 검사를 추가해야 합니다.

## 실시간 이동

- WASD / 방향키로 자유 이동 (그리드 없는 탑다운), 카메라가 플레이어를 따라감
- 마우스 커서 방향으로 캐릭터가 회전 (이동 방향과 별개로 조준 가능)
- Socket.io로 접속/이동/공격/연결 해제를 모든 클라이언트에 실시간 브로드캐스트
  - 위치 동기화는 클라이언트가 주기적으로(50ms) 전송 → 다른 클라이언트는 보간(lerp)해서 부드럽게 표시

## 알려진 한계 / 다음 단계 (예정)

- 몬스터는 배회만 하고 플레이어를 공격하지 않음 — 플레이어 피격/사망/리스폰 시스템 없음
- 몬스터 종류가 3종뿐, AI는 단순 배회뿐 (추격/도주, 보스 등은 미구현)
- 근접 공격은 장애물의 시야선(line of sight)을 고려하지 않음 — 나무 뒤 몬스터도 사거리 안이면 맞음
- 이동/좌표는 여전히 클라이언트를 대부분 신뢰 (물 진입 제한도 클라이언트 판정이라 우회 가능함).
  단, 공격 판정과 상점 거리 검증은 서버가 authoritative하게 함
- 실제 도트 스프라이트/타일셋 이미지 없음 (색상 도형 placeholder, `pixelArt` 렌더링 모드만 켜둔 상태)
- 배(보트) 아이템처럼 "레벨 10 이상에서 정상 속도로 항해" 하는 수단 없음 — 지금은 그냥 감속만 적용
- 로그인/계정 없음 — 같은 브라우저가 아니면 캐릭터가 이어지지 않음
- 관리자 패널에 인증 없음 (베타 전용, 위 "관리자" 섹션 참고)
